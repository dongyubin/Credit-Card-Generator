# 内容分析

- **内容类型：** 面向开发者和 QA 工程师的项目 README，同时承担产品介绍、SEO 落地和本地开发文档三种用途。
- **目标读者：** 支付集成开发者、QA 测试人员、学生以及需要虚拟信用卡测试数据的用户。
- **核心搜索意图：** 免费信用卡生成器、虚拟信用卡号、测试信用卡号、Stripe 测试卡、PayPal Sandbox 测试卡、Mastercard 测试卡、批量测试数据导出。
- **语气：** 简洁、专业、可信，优先说明实际用途和安全边界，避免夸大或暗示真实支付能力。
- **关键区分：** 随机生成的虚拟卡用于 UI、格式和数据测试；服务商发布的测试卡用于对应 Sandbox 或测试模式。两者不能混为一谈。
- **格式要求：** 保留原文标题层级、表格、列表、代码块、图片和链接；命令、路径、产品名及技术名词不翻译。
- **一致术语：** dummy card 译为“虚拟卡”；provider-published test card 译为“服务商发布的测试卡”；Sandbox 保留英文并按语境补充“测试环境”；payment gateway 译为“支付网关”。
